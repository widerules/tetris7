trait SpecDasher
extends AnyRef
Trait that enables a more concise expression of Spec describers and examples by providing an implicit conversion that adds “dash methods” (-- and -) to String. A string followed by a double dash (--) denotes a describer, and results at runtime in an invocation of describe. A string followed by a single dash (-) denotes an example, and results at runtime in an invocation of it. Here's an sample Spec written in “dashes” style: 
 import org.scalatest.Spec
 import scala.collection.mutable.Stack

 class StackDashSpec extends Spec with SpecDasher {

   "A Stack" -- {

     "should pop values in last-in-first-out order" - {
       val stack = new Stack[Int]
       stack.push(1)
       stack.push(2)
       assert(stack.pop() === 2)
       assert(stack.pop() === 1)
     }

     "should throw NoSuchElementException if an empty stack is popped" - {
       val emptyStack = new Stack[String]
       intercept[NoSuchElementException] {
         emptyStack.pop()
       }
     }
   }
 }

Running StackDashSpec from within the Scala interpreter will yield: 

 scala> (new StackDashSpec).execute()

You would see: 

 A Stack
 - should pop values in last-in-first-out order
 - should throw NoSuchElementException if an empty stack is popped

Note: This trait is deprecated and may be removed in a future release of ScalaTest. I included it in 0.9.4 so I could get informed feedback from users as to whether you think this should be included in the ScalaTest API. If it gets removed, then those who like this style of code can always just do this themselves, so it isn't that risky to use. But bear in mind it may go away.
